# MarTech + AdTech Operator Syllabus

_Format: Harvard-style case + theory + practical lab_
_Duration: 10 modules (4–6 weeks accelerated or 10 weeks standard)_

## Learning outcomes
By the end, you will be able to:
1. Design a modern MarTech/AdTech architecture for a growth team.
2. Choose and integrate leading tools without creating brittle stacks.
3. Build privacy-safe tracking and activation workflows.
4. Run measurement systems that support real budget decisions.
5. Operate a practical experimentation loop across creative, channels, and lifecycle journeys.

---

## Module 1 — Industry Map and Operating Model
**Theory:** Ecosystem map, stack archetypes, build-vs-buy logic, org design.  
**Practical:** Audit current stack into core/edge/legacy categories.  
**Harvard case prompt:** You inherit a 27-tool stack with poor attribution and rising CAC. Cut complexity without slowing growth.

## Module 2 — Customer Data Foundations
**Theory:** Event taxonomy, identity resolution, first-party data strategy, governance.  
**Practical:** Create an event schema + identity map for web/app/server.  
**Harvard case prompt:** Data team and marketing team disagree on source-of-truth for conversion events.

## Module 3 — CRM, MAP, and CDP Architecture
**Theory:** CRM vs MAP vs CDP responsibilities, orchestration patterns, composable CDP.  
**Practical:** Design a target architecture using one cloud CRM + one activation layer.  
**Harvard case prompt:** Enterprise leadership wants one platform; teams need flexibility and speed.

## Module 4 — AdTech Pipes and Programmatic Mechanics
**Theory:** DSP/SSP/exchange/ad server, auctions, supply-path transparency, fraud controls.  
**Practical:** Draw a full bid-request path and annotate risk points.  
**Harvard case prompt:** Performance drops after shifting spend to open exchange inventory.

## Module 5 — Privacy, Consent, and Signal Resilience
**Theory:** Consent mode, server-side collection, legal/regulatory constraints, data minimization.  
**Practical:** Build a consent-state matrix and implementation checklist.  
**Harvard case prompt:** EEA expansion requires compliant measurement with minimal performance loss.

## Module 6 — Measurement Stack: Attribution, MMM, and Experiments
**Theory:** MTA limits, incrementality, MMM, triangulation strategy.  
**Practical:** Build a measurement decision tree for budget allocation.  
**Harvard case prompt:** Platform-reported ROAS conflicts with finance-level revenue outcomes.

## Module 7 — Lifecycle and Journey Orchestration
**Theory:** Lifecycle stages, trigger design, personalization logic, fatigue controls.  
**Practical:** Implement a 3-journey lifecycle blueprint (activation, nurture, winback).  
**Harvard case prompt:** High trial signups, weak activation and retention.

## Module 8 — Creative Intelligence and AI-assisted Campaign Ops
**Theory:** Creative systems, prompt pipelines, testing matrices, guardrails for AI-generated assets.  
**Practical:** Build a creative test matrix and winner-promotion policy.  
**Harvard case prompt:** Team scales creative volume 5x but CPAs worsen.

## Module 9 — Tooling Decisions and Vendor Evaluation
**Theory:** TCO, lock-in analysis, integration complexity, security/procurement gates.  
**Practical:** Score 3 vendor options with a weighted decision matrix.  
**Harvard case prompt:** Board pressure to consolidate vendors vs operator need for best-of-breed.

## Module 10 — Capstone: Operating a Full-Funnel Revenue Engine
**Theory:** KPI tree design, operating cadences, cross-functional governance.  
**Practical:** Build a 90-day execution roadmap with metrics and ownership.  
**Harvard case prompt:** New CMO has one quarter to prove marketing efficiency and growth quality.

## Leading product familiarity targets
- **CRM/MAP:** Salesforce, HubSpot, Braze, Iterable, Adobe Campaign
- **CDP/Data:** Segment, mParticle, Tealium, RudderStack, Snowplow
- **Warehouse/Activation:** Snowflake, BigQuery, Databricks, dbt, Hightouch/Census
- **Media/Ad buying:** Google Ads, DV360, Meta Ads, The Trade Desk, Amazon Ads
- **Measurement:** GA4, Amplitude, Mixpanel, Optimizely/Statsig, MMM tooling
- **Privacy/Consent:** OneTrust, Consent Mode, server-side GTM patterns

## Assessment model
- Module quiz/checkpoint (knowledge)
- Lab artifact (applied skill)
- Case memo (decision quality)
- Final capstone operating plan (execution readiness)
